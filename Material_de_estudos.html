Algoritmo

Quando vamos exercer uma tarefa, nós pensamos em todos os passos necessário para concluir ela. Para os computadores segue na mesma lógica, precisamos falar todos os passos necessário para que ele faça alguma coisa, isso se chama entrada de dados e processamento dele, e o que ele entregar é a saída. Toda a descrição de ações que passamos, fazendo com que a maquina obedeça, resultando uma sucessão de passos finitos para atingir um objetivo, se chama algoritmo, ou seja, todo o seu codigo fica em um algoritmo.

Variáveis e Constantes

Mas o que são as variáveis e constantes?

As váriáveis são utilizadas para guardar algum tipo de informação na memória RAM. Toda vez que declaramos uma variável, estamos reservando um espaço na memória para uso futuro, sofrendo alterações a partir com os dados do usuário.

Já as constantes, como o nome já diz, não sofrem alterações no decorrer do algoritmo. Serve para apresentar resultados estaticos ao usuário ou controlar internamente valores que nunca serão alterados.

Tipo de dados(variáveis e constantes)

Os dados utilizados no computador devem ser abstraídos para serem processados. Podem ser categorizados em três tipos primitivos:
-Numéricos(inteiros ou reais);
-Caracteres(alfabeticos ou alfanuméricos);
-Lógicos(verdadeiros ou falsos).

Dados numéricos inteiros
São números positivos e negativos pertencentes ao conjunto numérico inteiros, excluindo qualquer numero fracionado. Ex: 35, 0, -197, 2168, -5087

Dados numéricos reais
São dados positivos e negativos pertencentes ao conjunto de números reais.Ex: -87.2, 0, 67, 19.002, 0.7

Dados caracteres
São dados delimitados por aspas inglesas(""). Esse tipo é poder conter qualquer coisa tipo de caracter (A até Z), (0 a 9), símbolos, caracter especial, etc. Ex:"Av. Brasil, 1225", "(11) 99472-9037", "Masculino", "87090-120"

Dados lógicos
São lógicos os dados com valores binários como sim e não, verdadeiro e falso, 1 e 0, entre outros, e apenas um dos valores pode ser escolhido. Utilizaremos os valores aqui como VERDADEIRO/FALSO.O tipo de dado lógico também é conhecido como booleano — homenagem ao matemático George Boole.

Identificadores

Para que possamos utilizar variáveis, é necessário atribuirmos uma identificação (nome dado à variável).Os identificadores devem obedecer a algumas regras básicas e outras específicas (de acordo com a linguagem de programação utilizada).
Regras Básicas:
Não utilizar caracteres especiais (@, %, $, #, etc.), exceto o underline (_);
Não iniciar identificadores com números;
Não utilizar palavras reservadas (VERDADEIRO, INICIO, LEIA, ESCREVA, etc.);
Não utilizar caractere de espaço para unir palavras (utilize o underline).
Dica Prática: Imagine que o identificador é a "etiqueta" de uma caixa. Para o computador não se confundir, essa etiqueta precisa ser clara e seguir essas regrinhas de etiqueta digital.

Exemplos VÁLIDOS
Estes nomes seguem todas as regras e podem ser usados sem problemas:
nome_usuario (Usa underline para separar palavras);
idade2 (Números são permitidos, desde que não no início);
valorTotal (Letras maiúsculas e minúsculas são aceitas);
_status (O underline é o único caractere especial permitido no início).
Exemplos INVÁLIDOS
Estes nomes causariam erro no seu código:
2ano — Erro: Começa com um número;
nome usuario — Erro: Contém um espaço vazio;
valor% — Erro: Contém o caractere especial %;
ESCREVA — Erro: É uma palavra reservada (comando do sistema);
e-mail — Erro: O hífen - é considerado um caractere especial/operador matemático; o correto seria e_mail.

Expressões aritiméticas e lógicas

Uma operação muito comum na programação de computadores é a expressão aritimética para estabelecer processamento matemático e lógico para efetuar comparações. Sendo os operadores mais usados a multiplicação, representado pelo *, divisão, /, adição, +, subtração, -.
Uma expressão relacional é a comparação entre dois valores do mesmo tipo, utilizando variáveis e constantes. São eles: <, menor que, <=, menor ou igual que, >, maior que, >=, maior ou igual que, <>, diferente que, =, igual.O sinal de atribuição permite que seja atribuido um valor a variavel onde a natureza desse valor seja compativel com o tipo da variavel. Ex: A fórmula A=B+C é representado em algoritmo por: A<- B+C, dizemos que a variavel A recebe B mais C.

Estrutura condicional

Desvio Condicional

Também conhecido como Tomada de Decisão e Estrutura de Controle, tem como objetivo desviar o caminho de execução dos comandos de acordo com condição ou condições impostas juntamente com uso de operadores (quando necessário).

Operadores relacionais Operadores Logicos
> maior e conjunção
>= maior ou igual ou disjunção
< menor não negação
<= menor ou igual
= igual
<> diferente

Há alguns tipos de Desvio Condicional:

  • Simples (um único caminho)
  • Composto (dois caminhos)
  • Encadeado (diversos caminhos - um dentro do outro)
  • Múltipla escolha (diversos caminhos separados)

Desvio Condicional Simples

O desvio condicional simples é contido de uma ou mais condições e um único caminho apenas caso a condição (ou condições) seja satisfeita.


                Se (condição/condições) então
                    comandos...
                Fimse

Desvio Condicional Composto

O Desvio Condicional composto é contido por uma ou mais condições e dois caminhos a seguir, ou seja, se verdadeiro toma-se o caminho ENTAO, se falso, toma-se o caminho SENAO.


                Se (condição/condições) então
                    comandos...
                Senao
                    comandos...
                Fimse

Desvio Condicional Encadeado

Tanto no desvio simples quanto no composto, podemos utilizar, se necessário for, desvio condicional encadeado, ou seja, a resultante de uma condição (ENTAO ou SENAO) pode ser outra condição. Exemplo:


                Se (condição/condições) então
                    comandos...
                Se (condição/condições) entao
                    comandos...
                senao
                    comandos...
                Fimse
                Fimse

Desvio Condicional de Múltipla Escolha

Quando for necessário fazer vários testes para uma variável, podemos utilizar este tipo de desvio. Exemplo


                Escolha (x)
                Caso 1
                    comandos...
                Caso 2
                    comandos...
                Fimescolha

Laços de repetição

O que são laços de repetição?

Estruturas que permitem executar um conjunto de instruções diversas vezes.

Quando usar os laços de repetições?

Utilizamos LAÇOS DE REPETIÇÃO quando precisamos repetir tarefas até que:

  • uma condição (ou conjunto de condições) seja atendida.
  • ou por um número definido de vezes.

Vantagens do uso do laço de repetição

  • Evitam repetição de código
  • Facilitam manutenção e clareza
  • Podem trabalhar com quantidade fixa ou variável de repetições.

Tipos de laços de repetição:

  1. REPITA ... ATE - executa primeiro, testa depois (pelo menos uma vez)
  2. ENQUANTO...FAÇA - testa primeiro, executa depois (pode não executar nenhuma vez)
  3. PARA - Repetição controlada por um contador (número conhecido de repetições)

Repita...ATE

Na condição (ou condições) utilizaremos os operadores relacionais (>, >=, <, <=, =, <>) e os operadores lógicos (e, ou, não).


                Repita
                codigo...
                Ate (Condição)

A condição é o final do laço de repetição, ou seja, executa primeiro, testa depois.

Enquanto...FACA

                Enquanto (condição) Faca
                codigo...
                Fimenquanto

A condição é no início do laço, ou seja, testa antes, executa depois.

Para


                Para (inicio) de inicio ate fim passo (incremento) Faca
                codigo...
                Fimpara
  1. Variável de controle (inteiro)
  2. Valor inicial da variável de controle
  3. Valor final da variável de controle
  4. Passo do incremento da variável de controle (opcional)

Resumo

REPITA..ATE e ENQUANTO..FACA, embora possam ser utilizados para laços de quantidade definida, são ideais para validação de dados, porque eles repetem até que uma condição lógica seja satisfeita.
PARA é indicado para repetições que sabemos um determinado número de vezes ao qual o laço será executado.

Exercícios

Está na hora da pratica! Aqui estão alguns exercícios

  1. Solicitar o nickname e a idade de um jogador. Informar se ele pode participar de um campeonato que exige idade mínima de 16 anos.
  2. Solicitar um valor numérico e apresentar a mensagem: “Valor positivo”, “Valor negativo” ou “Zero”.
  3. Solicitar dois valores distintos e informar qual é o maior.
  4. Solicitar o mês (em número) e apresentar seu nome por extenso (ex.: 1 → janeiro).
  5. Solicitar o ano de nascimento (ex.: 2005) e informar se o ano é bissexto ou não.
  6. Solicitar o nome e a idade de 3 pessoas (utilizando a estrutura REPITA...ATÉ).
  7. Solicitar o nome e a idade de 3 pessoas (utilizando a estrutura FAÇA...ENQUANTO).
  8. Solicitar o nome e a idade de 3 pessoas (utilizando a estrutura PARA).
  9. Apresentar os números de 5 a 20, um por linha.
  10. Solicitar o nome, idade, sexo e salário de 3 pessoas e informar se homens ganham mais, mulheres ganham mais ou se é impossível determinar com os dados coletados.
